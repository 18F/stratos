To deploy your app make a code change in your repository, this will automatically trigger the delivery pipeline:

Add a manifest.yml to your source code or update your existing one by copying & pasting the code below:

We have:
  • reserved a name/route for your app
  • provisioned and bound services
  • created a temporary app in the gallery
  • created a delivery pipeline
applications:
  services:
    meteor-cloudant:
      host: meteor-cloudant-rocks
      name: meteor-cloudant-rocks
      descriptions: this application demonstrates how to use Cloudant with Meteor
      memory: 512M
      buildpack: http://github.com/cloudfoundry-community/cf-meteor-buildpack.git
      env:
        MONGO_URL: nope
  declared-services:
    meteor-cloudant:
      label: CloudantNOSQLDB
      plan: Shared

If you have updated your manifest already, any other code change will trigger a deployment.
